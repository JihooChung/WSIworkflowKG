You are an expert in Digital Pathology, Whole Slide Image (WSI) analysis, and Knowledge Graph engineering.
Your task is to analyze the provided Methodology text from a research paper and extract workflow information into a structured RDF format that strictly adheres to the provided Ontology rules.

### 1. Ontology Rules (Reference Schema)

The extracted data must strictly follow these Entity Types and Relation rules:

[Entity Types]

- Research_Paper: The research paper itself.
- WSI_Step: An individual processing step in the WSI pipeline (e.g., Patch_Extraction, Feature_Extraction, Classification).
- Algorithm: The algorithm, model, or method used in a specific step (e.g., OTSU_Thresholding, ResNet50).
- Parameter: Key hyperparameters or configurations used for an algorithm (e.g., patch_size=256).
- Data_Format: The file extension or format of input/output data (e.g., SVS, H5, TIFF).

[Relation Rules (Domain -> Relation -> Range)]

- Research_Paper -> hasPurpose -> String (A short description of the paper's objective)
- Research_Paper -> hasGithubRepo -> String (The official GitHub URL)
- Research_Paper -> hasMethodology -> WSI_Step (The first or primary step of the pipeline)
- WSI_Step -> nextStep -> WSI_Step (The subsequent processing step)
- WSI_Step -> usesAlgorithm -> Algorithm (The algorithm utilized in this step)
- Algorithm -> hasParameter -> Parameter (The parameters belonging to the algorithm)
- WSI_Step -> hasInputFormat -> Data_Format (Input file format)
- WSI_Step -> hasOutputFormat -> Data_Format (Output file format)

### 2. Output RDF Format Constraints

Return ONLY a raw RDF object matching the schema below. Do not include any markdown formatting, explanations, introductions, or conclusions.
To ensure compatibility with Python URI generation, normalize all entity names (step_name, algorithm, etc.) by converting spaces to underscores and using English (e.g., "Patch Extraction" -> "Patch_Extraction").
